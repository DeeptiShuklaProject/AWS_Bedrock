# 07_Chapter_configuration_files

## 🎯 Learning Objectives
In this chapter, you will learn how to:
- Configure local environment variables in a `.env` file.
- Manage dependencies and metadata in a `pyproject.toml` file.
- Configure deployment settings in `bedrock_agent_core.yaml`.
- Enforce security best practices for project configurations.

### Importance of This Chapter
Configuration files manage environment properties and dependencies, allowing you to deploy the same codebase across development, staging, and production environments without changing the code.

---

## 📦 Technical Terms Explained

> **📦 Technical Term Explained**
>
> **Term:** Environment Variables
>
> **Simple Explanation:** Environment variables are dynamic key-value pairs stored by the operating system that can be accessed by running processes to customize application behavior without modifying the code.
>
> **Why do we need it?** To store configuration settings and sensitive keys outside of your source code, preventing credential leaks.
>
> **Where is it used?** In shell environments or configuration files loaded by application processes.

---

> **📦 Technical Term Explained**
>
> **Term:** `.env`
>
> **Simple Explanation:** A `.env` file is a plain text file used to define environment variables for local development.
>
> **Why do we need it?** It simplifies managing configuration values locally by loading them into the application's environment automatically at startup.
>
> **Where is it used?** Stored at the root of your project directory (and typically excluded from git version control).

---

> **📦 Technical Term Explained**
>
> **Term:** `pyproject.toml`
>
> **Simple Explanation:** `pyproject.toml` is a configuration file format used by modern Python projects to define packaging settings, build dependencies, and tool settings.
>
> **Why do we need it?** It standardizes project settings and metadata across different build systems.
>
> **Where is it used?** Stored at the root of Python projects.

## 📄 1. The Environment Variables File (`.env`)

The `.env` file stores local configuration properties and credentials. Create this file at the root of your project:

```ini
# Folder Location: agentcore-samples/.env

# The AWS region where your Bedrock models and resources are hosted
AWS_REGION=us-east-1

# The foundation model ID used by the agent
BEDROCK_MODEL_ID=anthropic.claude-3-5-sonnet

# Set to True to enable local testing mode (bypasses Cloud ECR builds)
AGENTCORE_LOCAL_MODE=True

# Injected by Cognito authentication inside the production gateway
COGNITO_USER_POOL_ID=us-east-1_abcdef123
```

## 📄 2. The Project Configurations (`pyproject.toml`)

The `pyproject.toml` file manages project metadata and package requirements. Let's analyze a standard configuration:

```toml
# Folder Location: agentcore-samples/pyproject.toml

[project]
name = "agentcore-samples"
version = "0.1.0"
description = "Production-grade Bedrock AgentCore Samples"
readme = "README.md"
requires-python = ">=3.11"

dependencies = [
    "strands-agents>=0.1.2",
    "boto3>=1.34.40",
    "bedrock-agent-core-starter-toolkit>=0.1.0"
]

[tool.uv]
package = true
```

### Key Property Explanations

- **`requires-python = ">=3.11"`:** Restricts installation to Python 3.11 or higher to ensure compatibility with the runtime libraries.
- **`dependencies`:** Lists the required packages that the build system will download and package into the deployment container.

## 📄 3. The Deployment Metadata (`bedrock_agent_core.yaml`)

This YAML file is generated by the CLI configure wizard and contains deployment metadata.

```yaml
# Folder Location: agentcore-samples/bedrock_agent_core.yaml

agent_name: "aws_show_and_tell_agent"
entry_point: "src/main.py"
runtime_settings:
  python_version: "3.11"
  execution_role_arn: "arn:aws:iam::123456789012:role/AgentCoreExecutionRole"
  memory_id: "agentcore-memory-user-db-987"
```

### Key Property Explanations

- **`entry_point: "src/main.py"`:** Points to the main file executed when the container starts.
- **`execution_role_arn`:** The IAM role assumed by the runtime VM to access AWS resources.
- **`memory_id`:** The DynamoDB index key used to fetch and store user profiles.

## 🛠️ Common Mistakes & Troubleshooting
- **Mistake:** Accidentally committing your `.env` file to your Git repository, exposing sensitive credentials.
  - **Resolution:** Add `.env` to your project's `.gitignore` file.
- **Mistake:** Specifying an invalid value for `entry_point` in your configuration.
  - **Resolution:** Verify that the file path matches the location of your main execution script (e.g. `src/main.py`).
