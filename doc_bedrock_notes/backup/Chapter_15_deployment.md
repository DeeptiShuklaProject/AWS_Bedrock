# Chapter_15_deployment

## 🎯 Learning Objectives
In this chapter, you will learn how to:
- Package your agent application in a lightweight Docker image.
- Configure `Dockerfile` and `.dockerignore` files.
- Compile container images using AWS CodeBuild build runs.
- Push images to Amazon Elastic Container Registry (ECR).

### Importance of This Chapter
For production environments, agents must be deployed as container images. The AgentCore Runtime pulls these images from Amazon ECR to boot the isolated microVM sessions.

---

## 📦 Technical Terms Explained

> **📦 Technical Term Explained**
>
> **Term:** Dockerfile
>
> **Simple Explanation:** A Dockerfile is a text document containing a sequence of commands and instructions that Docker executes to assemble a container image.
>
> **Why do we need it?** It automates the image compilation process, ensuring your application environment is built consistently every time.
>
> **Where is it used?** Stored at the root of your project directory.

---

> **📦 Technical Term Explained**
>
> **Term:** ECR (Amazon Elastic Container Registry)
>
> **Simple Explanation:** Amazon ECR is a fully managed Docker container registry that makes it easy for developers to store, manage, and deploy Docker container images.
>
> **Why do we need it?** It provides secure cloud storage for your container images, integrated with AWS deployment services.
>
> **Where is it used?** Used by the compute runtime to pull and launch agent containers.

---

> **📦 Technical Term Explained**
>
> **Term:** CodeBuild (AWS CodeBuild)
>
> **Simple Explanation:** AWS CodeBuild is a fully managed build service that compiles source code, runs tests, and produces software packages (like Docker images) ready for deployment.
>
> **Why do we need it?** It handles the build CPU overhead in the cloud, removing the need to compile large containers locally.
>
> **Where is it used?** Automatically invoked when deploying resources via CLI.

## 🧱 The Deployment Lifecycle

The deployment process follows these steps:

```
┌─────────────────┐       1. agent-core launch       ┌─────────────────┐
│  Developer PC   ├─────────────────────────────────>│  AWS CodeBuild  │
└─────────────────┘                                  └────────┬────────┘
                                                              │
                                                              │ 2. Compile Docker
                                                              │    & Push ECR
                                                              ▼
┌─────────────────┐       3. Pull ECR & Launch       ┌─────────────────┐
│AgentCore Runtime│◄─────────────────────────────────┤   Amazon ECR    │
└─────────────────┘                                  └─────────────────┘
```

1. **Packaging:** The developer runs `agent-core launch` to upload the code and configuration assets to S3.
2. **Compilation:** S3 triggers an AWS CodeBuild task. CodeBuild creates a Dockerfile, compiles the Python dependencies, and pushes the container image to Amazon ECR.
3. **Deployment:** The ECR image is associated with the Agent Core Runtime to host new session invocations.

## 📄 The Dockerfile Configuration (`Dockerfile`)

Here is a standard `Dockerfile` generated by the CLI configure wizard:

```dockerfile
# Folder Location: agentcore-samples/Dockerfile

# 1. Use the official slim Python runtime
FROM python:3.11-slim

# 2. Configure environment settings
ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1
WORKDIR /app

# 3. Copy dependency manifest and install packages
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 4. Copy application files
COPY src/ ./src/

# 5. Expose HTTP port for the listener
EXPOSE 8080

# 6. Define the start command
CMD ["python", "src/main.py"]
```

### Key Property Explanations
- **`FROM python:3.11-slim`:** Builds the image on a minimal Python base image to keep container sizes small and boot times fast.
- **`ENV PYTHONUNBUFFERED=1`:** Configures Python to send outputs directly to stdout, ensuring logs are captured by CloudWatch in real-time.
- **`EXPOSE 8080`:** Opens the port used by the `BedrockAgentCoreApp` listener to receive requests.

## 🛠️ Common Mistakes & Troubleshooting
- **Mistake:** Copying local virtual environments or temporary build directories into the image, increasing size and compile times.
  - **Resolution:** Use a `.dockerignore` file to exclude files like `.venv/` and `__pycache__/`.
- **Mistake:** Missing ECR permissions in your CLI role.
  - **Resolution:** Ensure your IAM role has the `ecr:InitiateLayerUpload`, `ecr:UploadLayerPart`, and `ecr:PutImage` permissions.
